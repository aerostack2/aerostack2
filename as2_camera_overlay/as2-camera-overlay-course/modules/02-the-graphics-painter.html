Projecting the Video

The drone's video isn't just "under" the 3D markers—it's actually part of the 3D scene. We draw it on a flat panel (a Quad) that fills the camera's view perfectly.

CODE 

void OverlayRenderer::createBackgroundQuad() {
  background_rect_ = new Ogre::Rectangle2D(true);
  background_rect_->setCorners(-1.0, 1.0, 1.0, -1.0);
  background_rect_->setRenderQueueGroup(Ogre::RENDER_QUEUE_BACKGROUND);
  Ogre::AxisAlignedBox aab_inf;
  aab_inf.setInfinite();
  background_rect_->setBoundingBox(aab_inf);
}

Create a 2D rectangle that will hold our video pixels...

Make it fill the entire screen (from top-left to bottom-right)...

Tell the engine to draw this rectangle FIRST, behind everything else...

Define a bounding box for the rectangle...

Make the box "Infinite" so the engine never hides the background, no matter where the camera looks.

Pasting Pixels (GPU Blitting)

When a new frame arrives, we don't just "copy" the image. We perform a blit operation, sending pixels directly to the GPU's texture memory for maximum speed.

Cleanup and Memory

The renderer must be a good citizen. Methods like destroyRenderTarget and destroyBackgroundTexture ensure that GPU resources are released when the node shuts down, preventing memory leaks.
