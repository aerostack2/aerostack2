Math Utilities

Drawing 3D markers on a 2D image is all about projection math. If the math is wrong by even 1%, the markers will appear to "drift" or float away from the drone's actual position.

💡
The "Secret Hack"

The 4x4 projection matrix is the most critical piece of math in the project. It warps the virtual 3D world to exactly match the distortions of the real physical camera lens.

The Lens Blueprint (Intrinsics)

Every camera has a "personality"—its focal length, its center point, and its sensor size. We capture this in the Intrinsics struct.

CODE 

struct Intrinsics {
  unsigned int width;  
  unsigned int height; 
  double fx, fy;       
  double cx, cy;       
};

Correcting Coordinate Chaos

Computer Vision and 3D Graphics don't speak the same language. visionToOgreRotation acts like a translator, swapping axes so that "Forward" in one system is "Forward" in the other.

1
CV Frame

Z is forward, X is right, Y is down.

2
Ogre Frame

Z is back, X is right, Y is up.

3
The Fix

Rotate the coordinates by 90/180 degrees to align them perfectly.

Stereo Offset (The Eyes of the Drone)

If the drone has two cameras (Stereo), applyStereoBaseline shifts the 3D world slightly left or right to match which "eye" is currently being used. Without this, the AR overlay would look "cross-eyed"!
