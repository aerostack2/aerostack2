Clever Tricks

How to make 3D rendering work on a flying robot

The RViz Cheat Code

Drawing 3D markers like grids and paths is hard. Instead of writing that code from scratch, the developers used a clever trick: they reused the exact same plugins that RViz uses.

🔌

Reuse, Don't Rewrite

Because it uses standard plugins, any 3D marker that works in RViz will work here too. It's like being able to use any brand of headphones with a phone because they all use the same plug.

🖼️

Virtual Buffer

The code tricks the graphics engine into rendering into a buffer instead of a window on a screen.

The Performance Hack: Render Scale

Drawing high-definition 3D graphics on a drone takes a lot of GPU power. To save energy and keep things fast, the developers added a "Scale" trick.

CODE 

scaled_k.width = std::max(1u, static_cast<unsigned int>(orig_w * render_scale_));
scaled_k.height = std::max(1u, static_cast<unsigned int>(orig_h * render_scale_));

cv::resize(cv_ptr->image, scaled_bg, cv::Size(scaled_k.width, scaled_k.height), 0, 0, cv::INTER_AREA);

// ... (after rendering) ...

cv::resize(rendered, rendered, cv::Size(orig_w, orig_h), 0, 0, cv::INTER_LINEAR);
PLAIN ENGLISH

Shrink the image size (e.g., to 50%) before we start drawing. This makes the math much easier for the drone.

Shrink the background photo to match our smaller drawing space.

The GPU now has 4x fewer pixels to worry about, making it much faster.

After we're done drawing, stretch the image back up to its original size so the user doesn't notice.

This trick is essential for maintaining a high frame rate while the drone is flying fast.
