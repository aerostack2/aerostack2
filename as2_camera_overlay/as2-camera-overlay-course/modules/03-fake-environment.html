Fake Environment

Standard RViz plugins are "spoiled"—they expect to live in a fancy windowed application with a mouse and a screen. Since we are running headlessly, we have to provide "Fake" versions of the environment they expect.

The Headless Cast Classes that trick RViz plugins
HeadlessRosNode Wraps the real ROS node to hide the headless nature.
HeadlessFrameManager Handles coordinate requests (TF) headlessly.
NoOpHandlerManager Safely ignores mouse clicks and keyboard events.
HeadlessDisplayContext The master "Fake RViz" shell.

The Master Illusionist

HeadlessDisplayContext is the most important class in this module. It implements the standard RViz DisplayContext interface but returns nullptr (Nothing) for any window-related request.

rviz_common::WindowManagerInterface *
HeadlessDisplayContext::getWindowManager() const {
  return nullptr; 
}

void HeadlessDisplayContext::queueRender() {
  render_dirty_.store(true);
  frame_count_.fetch_add(1);
}

Ignoring the Mouse

RViz plugins allow users to click 3D objects to see data. This uses a SelectionManager. Since we have no mouse, NoOpHandlerManager provides empty slots where plugins can "register" their click-handlers without crashing the program.

ℹ️
Architecture Detail

This "No-Op" pattern is a common trick in software engineering. By providing an object that does nothing instead of "No Object" (null), you prevent "Null Pointer Exceptions" and keep the code running smoothly.
