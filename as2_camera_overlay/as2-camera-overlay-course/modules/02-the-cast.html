The Three Roles

In the codebase, work is split between three main roles. Each one has a specific "superpower."

🤖

The Coordinator

OverlayNode acts like a project manager. It listens for camera images and hands them off to the artist.

🎨

The Artist

OverlayRenderer uses a 3D engine to paint markers and grids onto the video pixels.

🔌

The Plugin Manager

DisplayLoader lets us add new types of visuals without changing the main core code.

The Map of the Files

If you're asking an AI to help you fix a bug, you'll need to know which file handles which role.

src/
overlay_node.cpp The coordinator's brain. Handles ROS 2 setup.
overlay_renderer.cpp The artist's tools. Handles Ogre 3D commands.
display_loader.cpp The plugin organizer. Uses pluginlib to find display types.
headless_rviz.cpp The compatibility glue. Makes RViz components work without a screen.
frame_utils.cpp The math toolbox. Handles coordinate math and projection logic.
