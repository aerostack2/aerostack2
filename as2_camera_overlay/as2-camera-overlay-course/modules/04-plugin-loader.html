Plugin Loader

The "Secret Sauce" of as2_camera_overlay is that it doesn't try to reinvent the wheel. It uses a plugin loader to use the exact same code that RViz uses to draw grids and markers.

💡
Key Insight

By using pluginlib, we can load any RViz plugin dynamically. If a new type of marker is created for RViz, our node can use it instantly without any code changes!

The Plugin Manager

DisplayLoader is our dynamic manager. It takes a "Class ID" (like rviz_default_plugins/Grid) and instantiates the plugin code on the fly.

CODE 

bool DisplayLoader::loadDisplay(const DisplayConfig &cfg, 
                                const YAML::Node &properties) {
  auto display = loader_->createSharedInstance(cfg.class_id);
  display->initialize(context_);
  applyProperties(display->getRootProperty(), properties, logger_);
  display->setEnabled(true);
  displays_.push_back(display);
}

Filtering the "Bads"

Not every RViz plugin can run headlessly. Some plugins (like those that try to open a file-picker or a complex GUI) will crash if there's no screen. isExcluded acts like a bouncer at a club, checking IDs and turning away plugins that are "too graphical."
